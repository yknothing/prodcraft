We are building an audit-log capability for a B2B finance platform. Please turn the notes below into a first-release requirements set before architecture starts.

Stakeholder notes:
- Compliance: audit records for financial actions must be retained for 7 years.
- Security: only explicitly authorized roles may search or export audit logs.
- Operations: support engineers need fast search by account, actor, date range, and action type.
- Product: customers want to export filtered audit logs as CSV.
- Compliance: exported data must be traceable to who exported it and when.
- Security: audit records themselves must not be editable or deletable through the product.
- Platform: month-end load is heavy; search must remain responsive.
- Legal: some regions require data access to be justifiable and reviewable.
