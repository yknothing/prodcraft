# Seat and Guest Management Release-1 Requirements Draft

## Functional Requirements

1. The system should let admins invite external collaborators without giving them a standard full member seat by default.
2. The system should let admins control what invited guests can access.
3. The system should let admins revoke guest access when collaboration ends.
4. The system should make guest activity and status visible to admins.
5. The system should support approval or finance review where larger customers need it.

## Scope Notes

- Release 1 should emphasize guest collaboration over a full admin-controls redesign.
- Pricing and procurement concerns matter, but do not need to become the center of the first release.

## Open Questions

- What exact guest restrictions are mandatory in release 1?
- When should finance approval be required?
- How should larger governance-heavy accounts be handled if the basic guest model is not enough?

## Next Step

Use this draft as input for system design and acceptance criteria.
