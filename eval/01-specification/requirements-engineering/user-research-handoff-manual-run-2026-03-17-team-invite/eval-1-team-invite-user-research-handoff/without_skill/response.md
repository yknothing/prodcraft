# Team Invite Release-1 Requirements Draft

## Functional Requirements

1. The system should allow workspace owners to invite teammates by email.
2. The system should allow invited teammates to join the workspace through a simple acceptance flow.
3. The system should support role assignment for invited teammates.
4. The system should show invite status to the owner or admin.

## Scope Notes

- Release 1 should focus on lightweight team onboarding.
- Domain claim, SSO, and bulk provisioning can be addressed later if needed.

## Open Questions

- Should role selection happen before or after acceptance?
- When should domain claim or SSO become part of the onboarding path?
- When does bulk provisioning become necessary?

## Next Step

Use this draft as input for system design and acceptance criteria.
