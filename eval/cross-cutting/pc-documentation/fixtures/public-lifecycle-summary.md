# Public Lifecycle Summary

- `draft`: the skill exists, but the review packet is not yet strong enough to
  support broader maintenance or promotion decisions
- `review`: the skill has a review packet, a clear audience, and enough evidence
  to justify maintainers looking at it as a real contract
- `tested`: the skill has a concrete evidence trail and has cleared the smallest
  useful validation threshold for its current lifecycle stage

The repository uses lifecycle labels to track evidence maturity. The labels do
not replace the docs themselves; they tell maintainers how much confidence to
place in the docs and the surrounding evaluation artifacts.
