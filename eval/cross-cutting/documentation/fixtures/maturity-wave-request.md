# Maturity Wave Request

Prepare a short maintenance document for repository maintainers.

The document should explain:

- when a skill should move from `draft` to `review`
- what minimal evidence is needed before a skill can reach `tested`
- how to distinguish durable documentation from transient notes
- which files are authoritative when someone wants to understand the current
  skill maturity contract

Keep it focused. Do not write a changelog entry.
Do not produce a long note dump.
