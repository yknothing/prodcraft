# v1.1 Execution Board

> Status: proposed execution board for the `1.1` review-pool digestion wave.

## Scope

This board converts the version-wave roadmap into execution-ready work for the next promotion slice.

Wave rule:

- do not pull any new `draft` skill into `review`
- work only on the current public beta review spine:
  - `system-design`
  - `feature-development`
  - `code-review`

## Exit Rule

`1.1` is successful if all of the following become true:

1. each target skill has one current benchmark conclusion artifact
2. at least one target skill reaches `tested`
3. the remaining skills leave the wave with sharper blockers than "needs more evidence"

## Execution Order

1. `code-review`
2. `system-design`
3. `feature-development`

Why this order:

- `code-review` is the cheapest and most self-contained `1.1` slice because the benchmark lane already exists and the remaining blocker is now narrow review precision
- `system-design` is next because its benchmark design is already in place, but it still depends on a clean with-skill completion
- `feature-development` stays last because the rerun should happen only after the new minimal implementation fixture is in place
